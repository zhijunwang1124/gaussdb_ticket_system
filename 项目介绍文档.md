# GaussDB 工单问题分析系统 - 项目介绍

## 1. 项目概述

`GaussDB 工单问题分析系统` 是一个面向运维/技术支持场景的工单分析平台，核心目标是将原始工单数据与大模型智能分类能力结合，形成“可导入、可分析、可透视、可查询、可导出”的闭环流程。

项目当前采用**单体后端 + 内嵌静态前端页面**的结构：

- 后端：Spring Boot（Java 8）
- 前端：Vue 3 + Element Plus + ECharts（CDN 引入）
- 数据库：GaussDB（兼容 PostgreSQL 协议）


## 2. 核心业务流程

系统围绕三个主菜单展开：

- `工单分析`
  - Excel 批量导入工单
  - 配置分析列（支持是否顶层分类）
  - 发起大模型分析任务（SSE 实时进度）
  - 保存分析结果、导出结果
- `数据分析`
  - 按日期范围进行透视
  - 时间分组透视（年/月/周/日）
  - 分析列分组透视（自动选择扇形图/柱状图）
  - 支持点击图形后的级联分析
- `工单查询`
  - 查询已入库工单与分析结果

## 4. 关键功能说明

## 4.1 工单导入

- 通过 `POST /api/import/excel` 接口导入 Excel。
- 仅处理识别到合法工单号（以 `YW` 开头）的行。
- 导入时将工单字段映射到 `ticket` 表并执行 upsert。

## 4.2 分析列配置

- 通过 `analysis_column_config` 管理分析列元数据：
  - `column_key`
  - `column_name`
  - `description`
  - `allowed_values`
  - `is_top_level`
- 支持新增和更新分析列，并自动同步 `ticket_analysis_result` 物理列。

## 4.3 大模型分析任务

- 创建任务：`POST /api/analyze/tasks`
- 任务进度流：`GET /api/analyze/tasks/{taskId}/stream`（SSE）
- 停止任务：`POST /api/analyze/tasks/{taskId}/stop`
- 运行逻辑支持并发线程控制（`app.llm.concurrent-threads`）。

## 4.4 透视分析与图表

- 透视接口：`POST /api/pivot/query`
- 支持：
  - 日期范围过滤（到“日”）
  - 时间分组（`year/month/week/day`）
  - 分析列分组统计
  - 分类数大于 5 时推荐柱状图，否则扇形图
  - 图上显示数量与百分比
  - 级联透视（基于已选路径继续下钻）
- 前端采用 ECharts 提供图表展示、缩放、导出图片等交互能力。

## 4.5 结果保存与查询导出

- 保存分析结果：`POST /api/analysis-results/save`
- 查询原始工单：`POST /api/tickets/search`
- 查询已落库分析结果：`POST /api/persisted-tickets/search`
- 导出 Excel：`POST /api/tickets/export`


## 5. 数据模型（简要）

核心表：

- `ticket`：工单主表（包含起始日期、当前阶段、问题描述等业务字段）
- `analysis_column_config`：分析列配置表（含顶层分类标识）
- `ticket_analysis_result`：工单分析结果落库表（动态扩展列）
- `analyze_task` / `analyze_task_item`：分析任务与任务明细
- `ticket_analysis_snapshot`：分析快照（扩展用途）

初始化脚本在：`backend/src/main/resources/db/schema.sql`。

## 7. 运行与配置说明

## 7.1 环境要求

- JDK 8
- Maven 3.6+
- GaussDB / PostgreSQL 协议兼容数据库

## 7.2 关键配置

配置文件：`backend/src/main/resources/application.yml`

- 服务端口：`server.port`
- 数据源：`spring.datasource.*`
- 初始化 SQL：`spring.sql.init.schema-locations`
- LLM 配置：`app.llm.provider`、`app.llm.endpoint`、`app.llm.api-key`、`app.llm.model`

## 7.3 启动方式（后端）

在 `backend` 目录执行：

```bash
mvn spring-boot:run
```

启动后访问：

- [http://localhost:8080](http://localhost:8080)


## 8. 当前版本特点

- 已从导入流程中移除 URL 导入能力，仅保留 Excel 导入。
- 数据分析页面已升级为图表主导（时间透视 + 分组透视 + 级联透视）。
- 分析列支持“顶层/非顶层”分类策略，便于构建层级化分析路径。


## 9. 后续优化建议

- 增加统一鉴权（登录、角色、审计日志）。

## 10. 注意事项

- 当前 `application.yml` 中存在示例形式的敏感配置，请在生产环境使用安全方式管理密钥。
- 首次启动会执行 `schema.sql`，请确保数据库账号具备建表/变更权限。
